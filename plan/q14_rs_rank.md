# Q14 — 대장주 프록시: 전시장 RS 백분위 랭크 게이트

상태: **설계 확정 → 구현 → 스윕 대기** (2026-07-17 착수, 개선계획 §5 Q14 등재분의 실행 Phase)

## 1. 배경

- 규칙서 §3 1단계 "해당 테마의 1~2등 대장주"는 테마 분류 데이터 부재로 v1 백테스트
  범위에서 제외됐다(백테스트 계획서 §3 "범위제외"·§11 후속과제 2). 현행 구현의 1단계는
  `rs_6m > 0` 불리언(지수 대비 6M 상대수익 양수) 하나뿐 — 스크리너 F2도 동일.
- 편집형 테마 분류의 과거 재현은 편입 자체가 사후 정보라 룩어헤드 — 근사만 가능.
  개선계획 §5 Q14에 3개 선택지 등재(2026-07-17), 제안 (a) **전시장 RS 백분위 랭크
  게이트**부터 착수. (b) 업종 내 랭크·(c) 상관 클러스터링은 후순위.
- **Q13c(교체 규칙) 대비 우선 착수 근거**: ① Q13a가 자본 병목을 52→39로 이미 완화해
  Q13c의 잔여 상방은 한정적이고, 교체 규칙은 매도 대상 선정이라는 새 자유도(과최적화
  위험)를 도입한다. ② Q14는 전략 충실도 공백(규칙서 명시 항목의 미구현)을 메우는
  질 게이트라 퍼널 전체에 작용한다. ③ 게이트 강화는 후보 흐름 자체를 줄여 자본 경합을
  구조적으로 완화 — Q14 결과에 따라 Q13c의 필요성·규모가 재정의되므로 순서상 선행이
  합리적. ④ (a)안은 기존 가격 데이터만 사용, 완전 point-in-time, 코드 변경 국소.
- **방향 주의(등재 시 명기)**: P1~Q13은 게이트 완화(캡처 확대), Q14는 게이트 **강화**
  (캡처 감소 방향). 1차 지표를 캡처율 단독으로 두면 자동 기각되므로 평가 프레임을
  "캡처 손실 대비 질 개선"으로 사전 등록한다(→ Q14-7).

## 2. 결정 질문 (제안 기본값 — 임의 가정 금지 원칙에 따라 사전 등록)

**Q14-1. 랭크 대상 값** — 무엇을 랭크하나?
- **제안: 기존 `rs_6m`(6M 수익률 − 자기 시장 지수 6M 수익률, `return_diff`) 그대로.**
- 근거: 등재 문안("6M 상대수익의 횡단면 백분위")과 일치. 기존 게이트(`rs>0`)·동일일
  다중후보 정렬(RS 내림차순)과 산식 단일화 — 지표 이중화 없음. 지수 차감이 시장 간
  베타 차이를 부분 정규화해 전시장 통합 랭크(Q14-2)의 비교 가능성도 확보.
- 대안(기각): 원시 6M 수익률 — 지수 미차감이라 시장 국면에 따라 특정 시장이 상위를
  독식, 별도 지표 신설이라 이중화.

**Q14-2. 랭크 모집단 스코프** — 전시장 통합 vs 시장별(KOSPI/KOSDAQ) 분리?
- **제안: 전시장 통합(`all`)을 기본, 시장별(`market`)을 스윕 축으로 병행 실측.**
- 근거: 등재 문안 = 전시장. 다만 코스닥 강세장에서 통합 랭크가 코스닥에 쏠리는 왜곡
  가능성이 있어 두 스코프를 같은 스윕에서 비교해 데이터로 결정한다.
- 모집단 = **해당 런의 유니버스**(전 유니버스 런 = 전시장 2,580종목). 정밀 런·스윕
  모두 전 유니버스로 실행하므로 실질 전시장.

**Q14-3. 컷 임계값 X** — 상위 몇 %만 진입 허용?
- **제안: X ∈ {10, 20, 30} 스윕 + off(대조군).** 게이트 조건:
  `pct_rank(rs_6m) ≥ 1 − X/100` (pandas `rank(pct=True)`, 동점 average).
- 근거: 등재 문안의 스윕 값 그대로. 미너비니 실무 RS 70~90+ ↔ X=30~10 대응.

**Q14-4. 모집단 자격·스테일 한도** — 어떤 종목이 그날 랭크 모집단에 드나?
- **제안: 마스터 캘린더(런 캘린더) 축으로 각 종목 `rs_6m`을 as-of 정렬하되 전방채움
  한도 10세션.** 이력 부족(126d 미만, NaN)·최근 10세션 무거래(거래정지·상폐 잔존)는
  그날 모집단에서 제외되고 게이트도 탈락(기존 `RsFilter`의 None→False와 일관).
- 근거: 무한 as-of는 상폐 종목의 잔존값이 모집단을 영구 오염. 한도 10세션은 일시
  거래정지(수 일)는 살리고 장기 정지·상폐는 걸러내는 절충 — 상수로 구현(코드
  `STALE_LIMIT_SESSIONS = 10`), 민감도 이슈가 실측되면 그때 설정화.

**Q14-5. 적용 지점** — 하드 게이트 vs 정렬 우선순위?
- **제안: 5번째 하드 게이트로 추가**(트렌드·RS·시장·품질에 병렬, 모두 ≤d−1 기준 —
  랭크도 prev 시점 값). 기존 `rs>0` 게이트와 RS 내림차순 정렬은 **유지**(추가적).
- 대안(보류): 게이트 없이 랭크를 자본 배분 우선순위로만 사용 — Q13 잔여 병목 관점의
  변형이나 등재 문안은 게이트. (a) 기각 시 재상정 후보로 기록.
- 진단: 퍼널에 `gate_rs_rank_ok` 카운터, 게이트 분해에 `rs_rank_ok` 불리언 추가
  (`n_failed` 상한 7→8). 병목 판정 순서(`_FUNNEL_ORDER`)는 `gate_rs_ok` 뒤에 삽입 —
  off일 때 기존 라벨 불변.

**Q14-6. 스크리너 패리티** — 실전 일과에 어떻게 반영?
- **제안: 랭크 산출을 공유 모듈 단일 구현으로 두고 엔진·`screen_today.py`가 함께
  사용**(§6 이중 구현 금지 원칙). 스크리너는 활성 시 전 종목 1차 패스로 `rs_6m`을
  수집해 동일 백분위를 계산, `rs_rank_pct`·`rs_rank` 컬럼을 추가하고 `all_gate`에
  반영. **기본 off이므로 채택 전 일일 산출물은 불변.**

**Q14-7. 평가 프레임 (사전 등록)** — 무엇으로 채택/기각을 판정하나?
- **제안**: ≥4× 티어 캡처율은 감소를 전제하고, 판정은
  {**capture_sum_r**(캡처된 것의 질·규모), **기대값 R/트레이드**, **MDD**, **총수익**,
  스플릿(17-21/22-26) 일관성} 대 캡처 손실의 교환비로 한다.
- 채택 후보 기준(사전 등록): 캡처가 줄어도 capture_sum_r 또는 기대값R·MDD가 유의하게
  개선되고 방향이 두 스플릿에서 일관될 것. 전 지표 동반 악화면 기각하고 Q14(b)/(c)
  또는 Q14-5 대안(정렬 우선)을 재상정.
- 부차 관찰: all_pass 미체결 병목(39종목)의 변화 — 게이트 강화가 자본 경합을 얼마나
  구조적으로 줄이는지(Q13c 필요성 재평가 자료).

## 3. 구현 설계 (기본 off — 골든 불변이 수용 기준)

1. **설정** (`domain/config.py` `RsCfg` 확장, 기존 optional-키 관례):
   - `rs.rank_top_pct: float | null` — null=off(기본). (0,100) 밖이면 ConfigError.
   - `rs.rank_scope: "all" | "market"` — 기본 "all". 그 외 값 ConfigError.
   - `config/rules_v3-3.yaml`의 `rs:`에 두 키를 null/all로 추가(주석: Q14 스윕 전
     기본 off). `rulebook_version`은 v3-7 유지 — 버전은 승인 채택 시에만 올린다.
2. **공유 모듈** `src/oneil_bt/indicators/rs_rank.py`:
   - `build_rs_rank_table(rs_by_symbol, calendar, market_by_symbol, scope) -> DataFrame`
     — 종목별 `rs_6m` 시리즈를 캘린더 축 wide 프레임으로 정렬(ffill 한도 10세션),
     scope=all이면 행 단위 `rank(axis=1, pct=True)`, market이면 시장 그룹별 랭크.
   - as-of 조회 헬퍼(달력 비거래일 안전).
3. **엔진** (`engine/engine.py`): `_prepare`에서 활성 시 `_symctx`의 `rs_6m`로 테이블
   1회 구축(2,580×전기간 ≈ 수십 MB, 조합당 수 초). `GateResult`에 `rs_rank_ok` 필드
   추가(AND), `_entry_gates`에서 prev 기준 조회. 퍼널·게이트 분해·진단 CSV 헤더 확장.
4. **스크리너** (`scripts/screen_today.py`): 활성 시 1차 패스로 전 종목 rs 시리즈 수집
   (`RelativeStrength` 재사용, 가격 캐시라 2차 패스 무비용) → 공유 모듈로 L 시점
   백분위 → 게이트·컬럼 반영. off면 산출물 비트 동일.
5. **테스트**: config 3종 관례(기본값/생략=off/불량값 에러) + `test_rs_rank.py` 신설
   (경계·동점·NaN 이력·스테일 한도·market 스코프) + 엔진 게이트 차단·퍼널 카운터 +
   진단 테스트 상한 갱신(n_failed ≤8) + **골든 다이제스트 불변**(기본 off).
