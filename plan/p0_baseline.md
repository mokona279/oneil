## 작업 항목

### T1. 데이터 검증 (산출: `out/p0/data_validation.txt` + 실패 목록)

1. 배치 수집 리포트 취합 — 실패 종목 목록화, 원인 분류(빈 응답/정합 위반/기타), 재수집
   1회 시도 후 잔여 실패는 **명시적 제외 목록**으로 고정(조용한 누락 금지).
2. 전 종목 **엔진 로더 통과 스캔**: `CsvDataSource`로 전 종목 로드를 시도하는 스크립트.
   실패(스키마·정합·결측) 종목은 1의 제외 목록과 합산.
3. `meta.csv` 정합: prices/ 파일 전수 == meta 행 전수, market·listing_date 결측 확인
   (수집기 자기검증이 있으므로 재확인 수준).
4. 분포 통계: 종목 수, 종목당 세션 수 분포, 2015-10 시작(기존 상장) vs 상장일 시작(신규
   상장) 비율, 거래대금 100억↑ 종목 수(유효 유니버스 크기 감각).
5. **52주 미만 상장 종목**은 별도 처리 불필요 확인만: 52주 지표 `min_periods=252`가
   NaN을 주고 트렌드 템플릿이 자동 불통과 → 규칙서 §5 "IPO 유니버스 제외"와 일치.

### T2. 캡처 회귀 세트 추출 (산출: `out/p0/capture_set.csv`)

- 정의(Q8 확정): 백테스트 창(2017-01-02~2026-07-10) 안에서 **252세션 내 종가 기준
  +100% 이상** 상승을 달성한 적이 있고, 달성 구간에서 **20일 평균 거래대금 ≥ 100억**인 종목.
- 구현 위치: `src/oneil_bt/analysis/capture.py` (순수 함수 — 스윕·리포트에서 재사용).
  롤링 최소값 대비 현재 종가 비율의 벡터 계산으로 종목당 O(n).
- 출력 컬럼: `symbol, first_achieved(최초 +100% 달성일), max_multiple(창 내 최대 배수),
  turnover_ok, sessions`. 달성일이 창 밖(2017 이전)인 이력은 제외.
- 검수: 하이닉스·삼성전자가 세트에 포함되는지, 세트 크기가 상식선(수십~수백)인지 확인.
  임계(2.0×)에 세트가 민감하면 Q8 재상정.

### T3. 캡처 리포트 집계기 + 스윕 통합 (산출: `capture_report.csv`)

- 입력: 백테스트 산출물(trades.csv·entry_funnel.csv) × T2 캡처 세트.
- 종목별 출력: `entered(진입 발생 여부 — §2 합격 게이트), n_trades, sum_R(합산 pnl_r),
  contribution_pct(계좌 기여 %p), 미진입 시 퍼널 병목 단계(base 없음/단계/게이트명)`.
- 요약 출력: **캡처율**(세트 중 entered 비율), 캡처 종목 합산 R 분포.
- **스윕 통합**: `analysis/sweep.py` 결과 행에 `capture_rate`, `capture_sum_r` 컬럼 추가
  (상위 계획 §7 개선 1). 이것까지가 P0 — P1 캘리브레이션이 이 컬럼으로 줄을 세운다.
- 테스트: 합성 데이터로 집계 정확성 유닛 테스트(기존 테스트 스타일 미러링).

### T4. 전 유니버스 베이스라인 백테스트 (산출: `out/baseline_v3-3_2017_2026/`)

1. **파일럿 측정 먼저**: 200종목 표본으로 로드+지표 사전계산 시간·메모리, 일별 루프
   처리량을 재고 전체(2,579종목 × ~2,350세션) 소요를 외삽. 예상 병목: 지표 사전계산
   메모리(대략 1~3GB 추정)와 일별 루프의 파이썬 오버헤드.
   - 허용선을 넘으면(예: >4시간 또는 OOM) 개선 항목을 P0 내 태스크로 승격(심볼 청크
     로딩, 판정 루프 프로파일링). **규칙 로직은 건드리지 않는다.**
2. 본 실행:
   ```bash
   PYTHONPATH=src "C:/Users/mh.han/repos/daytrading/.venv/Scripts/python.exe" \
       -m oneil_bt.cli.run_portfolio \
       --price-dir data/prices --kospi data/kospi.csv --kosdaq data/kosdaq.csv \
       --meta data/meta.csv --rules config/rules_v3-3.yaml --costs config/costs.yaml \
       --start 2017-01-02 --end 2026-07-10 --cash 1e8 --out out/baseline_v3-3_2017_2026
   ```
   (`--symbols` 생략 = 전 종목. T1 제외 목록이 있으면 해당 종목 CSV를 뺀 목록을 넘긴다)
3. T3 집계기로 `capture_report.csv` 생성.

### T5. 베이스라인 성적표 고정 (산출: 상위 계획 문서 갱신 + `PROGRESS.md` 기록)

- 기록 항목: 총수익률·CAGR·MDD·승률·기대값R·트레이드 수·평균 노출·**캡처율·캡처 합산 R**,
  실행 소요·메모리.
- 재현성: 대규모 산출물이라 골든 해시 대신 `metrics.json` 스냅샷과 실행 명령·데이터
  범위(`fetch_report.json`의 range·universe_size)를 함께 기록한다.
- 서술 시 생존편향 문구 병기(상위 계획 §3.1).

## 리스크·주의

- **수집 실패 잔여 종목**: 소수면 제외 목록으로 진행(문서화 필수). 대량이면 수집기 문제 —
  P0 중단하고 원인 수정.
- **성능**: 파일럿 외삽으로 먼저 판단. 스윕은 P1부터라 P0에서는 단일 실행만 버티면 된다.
- **캡처 세트 임계 과적합**: 세트는 "정답지"가 아니라 회귀 감시 장치다. 임계 하나로 결과
  해석이 뒤집히면 정의를 단순화하는 쪽으로 재상정(Q8).
- **생존편향**: 세트·성적표 모두 현존 상장 종목 기준 — 절대치 해석 금지, 개정 전후 상대
  비교 전용.
