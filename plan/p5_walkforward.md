## §1 배경·목적

지금까지의 모든 규칙 채택(P1~Q13)은 **전체 창(2017-01-02~2026-07-10) 성과를 보고**
결정했다. 완화책으로 2017-21/22-26 분할 일관성을 병기했으나, 채택 판단 자체는
전체 창 인샘플이다. P5는 채택된 규칙 각각에 대해:

1. **한계효과의 시간 분산** — 규칙의 우위가 특정 연도(레짐)에만 몰려 있는가,
   여러 연도에 분산돼 있는가.
2. **선택 안정성(앵커드 워크포워드)** — "그 시점까지의 데이터만 알았다면 같은 값을
   골랐겠는가"를 2021~2026 각 시점에서 재현.
3. **패키지 최악 연도** — v3-7 전체 패키지가 v3-4(P2 이전) 대비 연 단위로 언제
   지는지, 최악 연도의 크기.

를 실측하고, 우위가 재현되지 않는 축은 **되돌림 재상정**한다(R3b는 P2 승인 조건에
"우위 미재현 시 null 되돌림을 기본 후보로 재상정" 기명시). 전 축 통과 시 규칙서
**v4 확정**으로 넘어간다.

**정직성 명시**: 진짜 아웃오브샘플 검증은 소급 불가능하다 — 모든 규칙이 이미 전체
창을 보고 채택됐다. P5가 측정하는 것은 "미래 성과의 증명"이 아니라 **"우위의 시간
구조가 과적합 신호를 보이는가"**(단일 연도 의존·후반부 소멸·선택 불안정)이다.

## §2 결정 질문 (제안 기본값 — 승인 전 가정 금지)

- **P5-1 검증 프레임**: 폴드별 재적합(문자 그대로의 워크포워드) vs **이산 축
  고정-암 + 연도 슬라이스** ← 제안. 근거: ① 재검증 대상이 전부 이산 on/off·이산
  값 축이라 폴드 내 "재적합"할 연속 파라미터가 없음 ② 엔진이 인과적(전체 런의
  t시점 상태 = t에서 멈춘 런과 동일)이므로 **전체창 정밀 런 1회의 연도 슬라이스 =
  해당 연도까지 운용한 계좌의 그 해 성과**와 동치 ③ 비용 9런×4분 vs 폴드별
  수십 런.
- **P5-2 암 구성 (9개)**: 기준 full(v3-7) + 단일 축 되돌림 7종 + 전체 되돌림
  legacy(v3-4 상당). 아래 §3 표. P1(R1·R2)은 재검증 의무 대상이 아니므로 전 암
  공통 유지(v3-4를 바닥으로 삼는 이유).
- **P5-3 슬라이스 정의**: 달력연도 2017~2026(2026은 07-10까지 부분). 연수익 =
  전년 마지막 세션 종가 자본 대비(2017은 초기자본). 연 MDD = 연내 러닝 피크 대비.
  실현 R·트레이드 수 = **exit_date 연도 귀속**. 발동 수 = rule_activations date 귀속.
- **P5-4 앵커드 선택 안정성**: 결정 시점 Y ∈ {2021..2026}. 축별로 "누적
  2017~(Y-1) 총수익 delta(채택값−legacy값)"의 부호 = 그 시점의 선택. 채택값과
  일치하면 선택 안정. 이어 Y연도 delta 부호와의 일치(선택이 보상받았는가)도 병기.
- **P5-5 축별 유지 판정 기준** ← 제안 기본값(최종 판정은 사용자 승인):
  - (a) **후반부 재현**: 2022~2026 누적 delta ≥ 0 (총수익 pp, ±2pp는 오차 중립).
  - (b) **단일 연도 의존 플래그**: 한 연도 delta가 전체 delta의 80%를 초과하면
    "단일 레짐 의존" 표기(자동 기각은 아니나 §3.3 추적 강화).
  - (c) **선택 안정성**: 2021~2026 앵커드 시점 과반에서 채택값 우세.
  - (a) 미충족 → 되돌림 재상정 후보. (b)(c)는 판정 표에 병기해 승인 질문에 올림.
- **P5-6 v4 확정 범위**: 전 축 판정 후 사용자 승인 시 rulebook_version v3-7→v4.0
  승격 + `oneil_strategy_v4_draft.md` 확정판 전환. 되돌림 발생 시 해당 키 변경 후
  확정. (이번 Phase 내 처리, 별도 결정 질문으로 승인.)
- **P5-7 비교 앵커**: full 암은 기존 candidate_B(out/q13)와 metrics 비트 일치
  확인(Q14 코드 골든 불변의 재검증 겸용). 창은 전 Phase와 동일하게
  2017-01-02~2026-07-10 고정(데이터는 07-16까지 있으나 앵커 비교 우선).

## §4 실행·분석 계획

1. 암 config 생성: `out/p5/make_arms.py` (정확 일치 단일행 치환 + assert).
2. 러너: `out/p5/p5_runs.cmd` — 9회 순차 `run_portfolio` (`--start 2017-01-02
   --end 2026-07-10 --cash 1e8`), 실패 시 FAIL 마커, 완료 시 DONE 마커. 분리
   프로세스(Start-Process Hidden)로 실행(세션 독립), RAM 제약상 동시 실행 금지.
3. 분석기: `scripts/walkforward_report.py` (영구 커밋 — F1 데이터 갱신 시 재실행
   의무가 있어 재사용). 입력 = 암 디렉토리들, 출력 = `out/p5/wf_yearly.csv`(암×연도
   지표), `wf_axis_delta.csv`(축×연도 delta), `wf_anchored.csv`(선택 안정성),
   콘솔 요약표.
   - 검증 불변식: full 암 연수익 복리 곱 = metrics.json total_return_pct(±0.01pp),
     각 암 exit-연도 R 합의 총합 = trades pnl_r 총합, 발동 연도 합 = 파일 행 수.
4. 판독: 축별 delta 표 + §2 P5-5 기준 판정 + 정밀 포렌식(필요 시 발동 트레이드
   단위 추적) → 트레이드오프 표 → 사용자 승인 → 반영.
