### 1.4 진단 (추론)

1. **진입 메커니즘이 단 하나** — "5~7주 성숙 베이스의 절대고점 피벗 돌파". 신고가를 자주
   갱신하는 종목은 베이스가 성숙할 틈이 없고(신고가마다 재무장), 성숙할 만큼 조정하면
   템플릿이 깨져 있다.
2. **11개+ 조건의 AND 게이트** — 표본이 5일뿐인데 독립적 게이트가 많아 매번 하나씩 걸린다.
3. **수축 게이트 변동성 비적응** — "피벗의 10%" 절대 기준.
4. **단계 영구 잠금** — 리셋 조건이 "직전 베이스 저점 하회"뿐이라 가장 강한 종목이 가장 먼저
   영구 금지된다.
5. **청산 후 복귀 경로 부재** — 60MA 청산 자체는 정당했으나(-28%·-39% 조정 회피) 복귀 수단이
   "새 5~7주 베이스 + 전 게이트"뿐. 베이스 없이 가는 구간(2026-04~06 +230%)은 참여 불가.
6. 결과적으로 현행 규칙은 "장기 횡보 후 턴하는 종목"(삼성)은 잡고 "쉼 없이 가는 1등 주도주"
   (하이닉스)는 못 잡는다 — 오닐 철학과 반대 방향의 바이어스.

## 2. 성공 기준 (확정)

- **합격 게이트 = 진입 발생**: 대상승 종목(§3.2 캡처 세트)에 전략이 최소 1회 진입했는가(이진).
  포지션 크기는 기존 사이징 규칙(1% 리스크·20% 상한)이 정하며 이 기준과 무관.
- **진단 지표(트레이드오프 판단용, 합격 기준 아님)**:
  - **종목당 합산 R** — 그 종목 트레이드들의 pnl_r 합. 사이징과 무관하게 "본 기회를 얼마나
    잘 탔는가"를 잰다. (시나리오 A=+2R vs B=+30R를 구분하는 지표)
  - **계좌 기여율(%p)** — 그 종목 트레이드 손익의 계좌 수익률 기여. 사이징 포함 최종 결과.
  - 포트폴리오: 총수익률·MDD·기대값R·승률·노출도.
- **MDD**: 고정 상한을 두지 않는다. 각 개정의 캡처 개선 ↔ MDD·기대값 악화를 표로 놓고
  트레이드오프로 결정한다(§6 Phase 종료 조건).

### 3.3 가드레일·과적합 방지

- **가드레일**: 개정 전후로 "기존에 잡던 트레이드"가 사라지거나 성격이 크게 바뀌면 원인 규명
  전에는 머지하지 않는다(삼성 2025-09 트레이드 등).
- **과적합 방지**: 파라미터 값 캘리브레이션(스윕)은 **하이닉스가 아니라 전 유니버스**로 한다.
  하이닉스는 "이 형태의 규칙이면 잡힌다/안 잡힌다"의 존재 증명으로만 쓴다.
- **워크포워드**: 최종 후보 config는 2017-2021 / 2022-2026 분할(또는 롤링)로 구간 안정성을
  확인한다(계획서 §11 후속과제의 선행 실행).
- 전 유니버스 실행 비용이 크면(측정 후 판단) 캘리브레이션은 층화 표본, 최종 검증만 전체로.
- **저표본 개정 추적 (P2 유래 상시 과제 — 2026-07-14 승인 조건, 사용자 지시)**:
  P2 채택분(R3b 리셋 N=12·Q11 클램프)의 실측 효과는 9.5년 표본에서 **각 1건**(하이닉스

## 4. 보강안 (규칙서 개정 단위)

각 항목: 현행 조항 → 측정된 문제 → 개정안. 값이 아니라 **형태**를 확정하는 것이 목적이며,
구체 수치는 전 유니버스 스윕으로 캘리브레이션한다.

### R1. 수축 게이트의 변동성 상대화 — §5 품질 3

- 현행: "돌파 직전 10거래일 장중 고저 범위 ≤ **피벗의 10%**".
- 문제: 절대 기준. 고변동 주도주는 구조적 통과 불가(5회 중 3회 차단, 실측 15.9~17.8%).
  2×ATR 게이트(품질 2)는 같은 날 전부 통과 — 두 게이트의 의도가 중복인데 단위만 다르다.
- 개정안: 레인지 기준을 종목 자체 변동성에 상대화 — **10일 레인지 ≤ k × ATR(14)** (k는 스윕
  캘리브레이션). 대안: 피벗 % 기준을 유지하되 상한을 `max(10%, k×ATR%)`로.
- 트레이드오프: 변동성 큰 종목의 "헐거운" 돌파가 통과 → 실패 돌파 증가 가능. 손절은 ATR
  연동이라 1회 손실은 1%로 유지됨(리스크는 빈도로만 증가).

### R2. 트렌드 템플릿 판정 보정 — §3 0단계

- **R2a. 200MA 상승 판정**: 현행 `MA200[D] > MA200[D-20]`(Q6 확정값). 장기 횡보 끝의 첫
  돌파에서 20일 룩백이 계단 후행(2025-06-05: 20일 -0.72%인데 일간 기울기 +). 개정안:
  룩백 단축(예: 5일) 또는 "MA200[D] > MA200[D-N], N∈{5,20} 중 하나 충족".
- **R2b. 52주 고가 밴드**: 현행 -15% 이내(v3에서 -25%→-15% 강화). 깊은(23~28%) 베이스의
  피벗은 구조적으로 전고점보다 ~20% 아래라 돌파 시점에 탈락(10/25·1/7 실측 -20%). 개정안:
  미너비니 원전 수준 **-25%로 환원**. (주의: 10/25·1/7은 사후적으로 손절 트레이드 — 이 완화의
  기대효과는 전 유니버스로 검증)
- 트레이드오프: 템플릿 통과 종목 증가 → 후보군 확대. 시장필터·품질 게이트가 뒤에서 거른다.

### R3. 베이스 단계 규칙 — §5 단계 카운트 4

- 현행: "1~3단계 진입 허용, 4단계 이상 신규 진입 금지". 리셋은 직전 베이스 저점 하회뿐.
- 문제: 다년 대추세에서 영구 잠금(1.2 표의 9/11·12/30). 오닐 원전도 후기 베이스는 "실패
  확률 높음"이지 금지가 아니다.
- 개정안(§5 Q5 — **병행 확정**):
  - **(a) 감액 진입**: 4단계+ 는 목표 비중 **절반**으로 진입 허용(1회 손실 -0.5%).
  - **(b) 리셋 조건 추가**: 마지막 유효 돌파 후 **N개월 이상 무돌파 + 새 유효 베이스
    (깊이 ≥ 20%)** 형성 시 카운트 1로 리셋 ("새 사이클" 판정). N은 스윕 축(제안 12개월).
- **병행 동작 방식**: 두 규칙은 서로 다른 지점에 걸린다. 새 베이스가 성숙(무장)할 때 단계를
  산출하면서 ① (b) 리셋 조건 충족 → 단계=1 (정상 사이즈), ② 미충족 → 기존 카운트 유지,
  진입 시점에 단계 ≤3이면 정상 사이즈, ≥4이면 (a)에 따라 절반 사이즈로 진입한다. 즉 (b)는
  "긴 횡보 뒤 새 사이클"을 정면으로 인정해 정상 참여시키고, (a)는 리셋감이 아닌 진짜 연속
  상승 중의 후기 베이스에 대한 안전망(참여하되 리스크 절반)이다.
  - 하이닉스 대입: 2025-06-05 베이스는 직전 유효 돌파(2023-10-17)에서 17개월+ 무돌파 후
    깊이 28% → (b) 리셋 → 단계 1, 정상 사이즈. 이후 9/11·12/30 베이스는 리셋 기점에서
    2·3단계가 되어 **감액 없이도 허용 범위**에 들어온다. (a)가 실제 발동하는 것은 리셋 없이
    4단계+에 도달한 다른 종목들 — 전 유니버스에서 빈도 측정.
- 트레이드오프: 후기 베이스는 실패율이 높다는 원전 경고 — 감액이 이를 반영. 빈도·MDD 증가분
  을 전 유니버스로 측정.

### R4. 진입 메커니즘 다변화 — §4·§5·§9 (가장 큰 구조 공백)

- **R4a. 컵위드핸들 핸들 피벗** — 규칙서 **내부 모순 해소**: §4는 "컵위드핸들이면 손잡이
  고점이 피벗"이라 하고, §5 정의 2는 "대체 피벗(핸들 고점) 불채택"으로 §4를 뒤집는다(구현은
  §5). 개정안: 핸들 피벗 채택 — 베이스 후반부(예: 마지막 1~3주)의 로컬 고점이 베이스 절대
  고점보다 낮으면 그것을 피벗으로 인정. 효과: 더 이른 진입 + 52주 고가 밴드와의 간섭 완화.
- **R4b. 추세 복귀 재진입 (50일선 되돌림)** — §9 "재진입은 새 돌파 신호가 났을 때만"의 "신호"
  확장. 자격: **직전 유효 돌파 이력 보유 + 트렌드 템플릿 통과** 종목이 60MA 청산 후, 가격이
  50MA(10주선) 부근에서 지지·반등(정의는 §5 Q6)하면 신규 트레이드로 재진입(1차만, ATR 손절
  재계산). 시나리오 A(+2R로 하차 후 10배 구간 방관)와 2026-04~06(+230% 미참여)이 직접 표적.
  - **손절 원칙(확정)**: 재진입은 **전량 청산 후 무포지션 상태에서만** 발동하는 신규
    트레이드다. 따라서 손절가 = **재진입 체결가(=새 평단) − 2×ATR(−10% 캡)** — 이전
    트레이드의 평단·손절가와 무관하다. 부분 청산(60MA 절반 매도) 후 보유 중인 상태의
    "절반 재매수"는 v1 범위 외(복잡도 대비 효익 불명 — 필요성이 확인되면 별도 상정).
- **R4c. 하이 타이트 플래그(선택)** — v3-3 "범위 외 · 명시적 불채택"의 **번복 여부 결정 필요**
  (§5 Q7). 8주 내 +100~120% 급등 후 3~5주 · 조정 ≤ 25% 패턴. 불채택 사유였던 "과열 제외와
  필연 충돌"은 HTF 요건 충족 시 과열 조항 (a) 면제로 해소 가능. 슈퍼주도주 전용 — 표본이
  귀해 전 유니버스에서 발생 빈도부터 측정.
- 트레이드오프: 진입 빈도·노출 증가(특히 R4b) → MDD 상승 예상. R4b는 엔진 이벤트 루프
  확장이 필요한 가장 큰 공사이므로 별도 Phase로 분리.

### R5. 시장 필터 — §2 (현행 유지 권고)

- 측정: 시장필터가 막은 10/25·1/7은 사후적으로 **-1R 손절 트레이드** — 필터가 정확히
  일했다. 60MA 후행성이 비용이 되는 사례는 이번 표본에 없음.
- 권고: **현행 유지.** 오닐 팔로우스루 데이(FTD) 도입은 전 유니버스 증거가 쌓인 뒤 후속
  연구로(§5 Q9).

백테스트 엔진과 **같은 컴포넌트를 재사용**해 실전 일과(규칙서 §8 루틴)를 자동화한다.
규칙 로직을 이중 구현하지 않는 것이 원칙 — 스크리너·매도 판정이 백테스트와 다른 답을 내면
그 자체가 버그다.

### F1. 데이터 업데이트 (부분 존재 — 보강)

- **현존**: `oneil_fetch` 재실행이 곧 업데이트다 — 증분 append + 수정주가 소급 감지(오버랩
  10일 비교, 불일치 시 자동 전체 재수집), `--end` 기본값 오늘, 유니버스 재산출로 **신규 상장
  자동 편입**(상장일부터 수집).
- **미비 → 보강 항목**:
  1. **KRX 세션 1시간 만료** 대응 — 전 유니버스 실행이 세션보다 길다. 이번 최초 수집은 배치
     드라이버(450종목×6, 프로세스별 재로그인)로 우회했으나, 이를 `oneil_fetch --batch-size`
     내장 옵션(배치 경계 재로그인)으로 정식화한다.
  2. **이름 조회 오버헤드**(~2,600콜 ≈ 20분/실행) — 유니버스 캐시 파일 재사용(`--universe-cache`,
     기본 7일 유효) 도입.
  3. **상폐 종목 처리**(Q12) — 유니버스 재산출에서 자연 제외되는 것을 meta에 반영(상태 컬럼
     또는 제외). 가격 CSV 보존 여부는 Q12 확정에 따름.
  4. **업데이트 요약 리포트** — 신규 편입/제외/소급 재수집/실패 종목을 실행마다 표로.

### F2. 매수 후보 스크리너 (신규)

없음 확인 — 진단 CSV(`entry_funnel` 등)는 백테스트에 결합된 산출물이고, "오늘 기준 후보"를
내는 독립 명령이 없다. 신규 CLI `run_screen --asof <날짜>`:

- 전 유니버스에 대해 판정일 기준으로: 유효 베이스 존재(피벗·깊이·기간·단계), 게이트별
  충족/미충족(트렌드 7조건 개별·RS·시장필터·과열·품질 4요건), `n_failed`.
- **전부 충족(돌파 대기) 종목**에 대해 매수 실행에 필요한 값 전부: 피벗(감시주문 트리거가),
  추격 상한(피벗+5%), ATR(14)·예상 손절가(피벗 체결 가정)·손절폭%, **목표 비중·1차 수량·
  금액**(계좌 평가액 입력), 2·3차 트리거가·상한가, 돌파일 거래량 기준(20일평균×1.5).
- **니어미스(`n_failed=1`) 목록** — 규칙서 §8 주말 루틴의 관심종목 등록·가격 알림 설정용.
- 출력: 콘솔 표 + CSV. 규칙서 §7 기록 의무(매수 이유·ATR·손절가·비중) 초안을 자동 생성.

### F3. 보유 종목 매도 판정 (신규) + 포지션 원장

없음 확인. 매도 판정에 필요한 입력은 **수량·평단만으로는 부족**하다:

| 판정 | 추가로 필요한 것 | 이유 |
|---|---|---|
| ① 손절 | **기록된 손절가** 또는 **진입일·피라미딩 체결 이력** | 손절가는 진입/피라미딩 시점 ATR로 굳는 경로 의존 값 — 오늘 ATR로 재계산하면 백테스트·규칙서와 어긋난다 |
| ② 60MA 이탈 | **이탈 진행 상태**(이탈일, 절반 매도 여부) | 3거래일 회복 카운트와 잔량 판정이 상태 의존 |
| ③ 시장 방어 | 이미 절반 축소했는지 | 반복 축소 방지 |
| 8주 룰 | **돌파일·진입가** | 3주 내 +20% 판정과 최소보유 56일 계산 |

→ 건별 수동 입력 대신 **포지션 원장 파일**(`portfolio_state.json`: 종목·트랜치별 체결일/
가격/수량·손절가·돌파일·피벗·이탈 상태·방어 축소 여부)을 두고, 매수·매도 실행 시 갱신한다.
F2 스크리너 출력이 원장 엔트리 초안을 만들어주므로 입력 부담은 낮다.

### F4. 일일/주말 운용 리포트 (신규 — F2·F3의 통합 소비자)

- **일일(저녁 5분, §8)**: 원장 읽고 → 보유 종목 ①②③·8주 판정 + **내일 실행할 주문 목록**
  (손절 매도/잔량 매도/방어 축소), 돌파 당일 종목의 거래량 1.5× 저녁 확인 → **2·3차 감시주문
  값**, 시장필터 상태(코스피/코스닥 vs 60·120MA), 리스크 거버너(연속 손절 카운트·중단 잔여일).
- **주말(30~60분, §8)**: F2 스크리너 전체 실행 + 보유 종목 ATR·손절가 점검표.

### 검토했으나 v1 범위 외

- **주문 API 연동** — 규칙서가 HTS 자동감시 주문 전제. 계산값 출력까지만.
- **실전-백테스트 정합 추적**(실체결가 vs 모델 체결가 → 슬리피지 실측 보정) — 실매매 개시
  후 데이터가 쌓여야 의미. 원장에 실체결가 필드만 미리 둔다.
- 부분 청산 후 "절반 재매수" — §4 R4b 손절 원칙 참조(범위 외).

## 7. 파라미터 민감도 스윕 개선 (기존 하니스의 한계 → 보강)

현행 `run_sweep`은 전 조합 데카르트 곱·단일 구간·포트폴리오 지표만·직렬 실행이다.
전 유니버스 캘리브레이션(P1~)에 쓰려면:

1. **캡처 지표 추가** — 조합별 출력에 캡처율·캡처 세트 합산 R을 포함(§3.2 집계기 재사용).
   지금은 총수익률로 줄 세우는데, 이번 진단이 보여주듯 **총수익률은 캡처 실패를 가리는
   지표**다(삼성만 잡아도 +40%).
2. **성능** — (a) 지표 관련 config가 불변인 조합 간 `IndicatorSet` 캐시 공유(현행은 조합마다
   재계산), (b) 프로세스 병렬화(조합 단위 독립·결정론이라 자명하게 병렬), (c) 캘리브레이션은
   층화 표본(시총·시장·변동성 층화) → 최종 후보만 전 유니버스.
3. **구간 안정성** — 조합별 지표를 2017-2021 / 2022-2026 분할로도 산출, **분할 간 랭킹
   상관**을 표시(한 구간에서만 좋은 조합 = 과적합 신호). 워크포워드의 스윕 내 통합.
4. **플래토 감지** — 축별 이웃 조합과의 성과 연속성 리포트. 고립된 스파이크 조합은
   채택 금지, 넓은 플래토의 중앙값을 채택.

## 8. 구현 Phase (각 Phase = 규칙서 개정 → config/코드 → 전 유니버스 검증 → 트레이드오프 표)

- **P0 — 베이스라인 고정 (규칙 변경 없음)** — 세부 계획: **`../plan/p0_baseline.md`**
  데이터 검증 → 캡처 세트 추출·집계기 → v3-3 전 유니버스 백테스트 → 베이스라인 성적표
  고정 + 실행 비용 측정. §7의 스윕 개선 1(캡처 지표)도 P0에 포함(P1 캘리브레이션이 바로 씀).
- **P1 — R1+R2 (게이트 보정)**: config 확장 + `base_quality`/`indicator_set` 소폭 수정.
  스윕으로 k·룩백 캘리브레이션(§7 개선 2~4 선행). 종료 조건: 캡처율 개선 & 가드레일 통과 &
  트레이드오프 표 승인.
- **P2 — R3 (단계 규칙)**: `stage_tracker` 리셋 조건 + 사이저 감액 경로 + Q11 손절 클램프.
  같은 종료 조건.
- **P3 — R4a (핸들 피벗)**: `base_detector` 확장. 같은 종료 조건.
- **P4 — R4b (추세 복귀 재진입)**: 엔진 이벤트 루프 확장 — 설계 문서 선행(트리거 정의 확정
  포함, Q6).
- **P5(선택) — R4c (HTF)**: Q7 재상정 후.
- **F트랙 (§6 실전 기능)**: 전략 개정과 독립 — F1은 수집 완료 직후, F2~F4는 P0의 컴포넌트
  (스크리너는 게이트 판정 재사용)가 갖춰지면 착수 가능. 단 F2~F4의 규칙 로직은 개정이
  확정된 버전을 따라가므로 **P2 이후 착수 권장**(이중 작업 방지).
- 마지막 — **워크포워드 검증** + 규칙서 최종 개정판 확정(버전 번호 결정) + README/PROGRESS 갱신.

각 Phase는 독립 커밋·독립 검증으로, 뒤 Phase가 승인 안 되어도 앞 Phase는 유효하다.

## 9. 리스크·한계

- **생존편향**(§3.1): 성과 절대치는 낙관 편향 — 캡처율 중심으로 해석.
- **단일 대세장 표본**: 2024-2026 반도체장은 극단 표본. 2017-2023(코로나 급락·2022 약세장
  포함)에서의 손익 특성이 개정의 실질 비용을 드러낼 것 — P0 베이스라인이 그 기준점.
- **게이트 완화의 비대칭**: 완화는 나쁜 진입도 늘린다. 1회 손실 -1%(감액 시 -0.5%) 구조는
  유지되므로 비용은 "손절 빈도·MDD"로 나타난다 — 트레이드오프 표가 이를 명시해야 한다.
- **R4b는 전략 성격 변화**: 돌파 전략에 추세추종 재진입이 섞인다. 규칙서 §9(복수 매매 금지)
  철학과의 정합을 개정 문언에서 명확히 해야 한다.
